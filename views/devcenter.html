Developer Center

Docs, endpoints, auth, pagination, and security notes.

Quick start

  1. Create an account (Sign up).
  2. Log in to establish a session.
  3. Call API endpoints under /api with cookies enabled.
  4. Admin-only endpoints require an admin role.
This project uses sessions (stored in MongoDB) and role-based middleware to protect endpoints.

Key endpoints

GET /api/bookspublic
POST /api/booksadmin
PUT /api/books/:idadmin
DELETE /api/books/:idadmin
POST /api/favoritesauth
DELETE /api/favorites/:idauth
GET /api/admin/usersadmin
PUT /api/admin/users/:id/roleadmin

Tip: All write operations are protected, and errors are handled safely.

Pagination

Use query params on books:

/api/books?page=1&limit=24

Response includes meta.total and total pages to build UI navigation.

Security notes

  • Passwords hashed with bcrypt.
  • Session cookies are HTTP-only; secure flag enabled in production.
  • Owner access enforced for user-owned resources (favorites).
  • Role-based access enforced for admin actions.
  • Secrets stored in environment variables.
